The Three Views

● Executive

Fleet Health Overview

One-screen status across all four environments. For leadership and the on-call lead.

  • 7 KPI tiles, env health matrix, severity timechart
  • Silent-server detection across Dev/Test/Stage/Prod
  • AG state changes & top noisy hosts
Open Executive view
● Operations

DBA Deep-Dive

Per-host drilldown for whoever the page just woke up. Tables, charts, and raw events on one screen.

  • Service crashes, severe errors, slow I/O, AG events
  • Backup outcomes by host & database
  • Custom telemetry: runaway-job Z-score & stuck sessions
Open Operations view
● Security

Authentication & Brute-Force

SOC-friendly view of who's hammering the SQL fleet, with off-hours and sa-account watchdogs.

  • Top attacking IPs & targeted accounts
  • Brute-force candidates (≥10 fails / 5 min)
  • Off-hours successes, host × hour heatmap

What it costs to ship

The whole stack rides on infrastructure already in place. The only ongoing maintenance is keeping the inventory CSV in sync with the SQL fleet.

New Software Licenses
$0 · uses existing Splunk + UF
New Agents on SQL Hosts
None · UF already installed
Initial Install
One person · half a day · single .spl upload
Ongoing DBA Workload
~5 min when a server is built or decommissioned
Search Head Capacity
Negligible at <200 hosts
Quarterly Tuning Review
~1 hour to confirm signal/noise ratio

What we're asking

Half-day install plus one sprint to swap mock inventory for real Dev/Test/Stage/Prod lists and wire two P1 alerts to PagerDuty.

30-day pilot: prod only · two P1 alerts (Service Crash + Backup Failure). If real-fire ratio is < 95% we tune; if it stays bad we roll back. Zero residual cost.
